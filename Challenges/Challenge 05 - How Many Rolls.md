# Challenge 05 - How Many Rolls
The object of this challenge is to create a program to determine how many die rolls it takes to get certain results.

## Mild 🌶️
Create a program that satisfies the following requirements:

* The user inputs an integer.
* The program randomly chooses an integer between 1 - 6 and adds it to a running total.
* If the total has been reached or passed, the program prints how many rolls it took to reach the total.
* The program prevents the user from typing invalid values and handles errors appropriately.
* The program only moves forward if input is valid.

## Medium 🌶️🌶️
Create a program that satisfies the following requirements:

* The user inputs an integer.
* The user inputs how many times the simulation should run.
* The program randomly chooses an integer between 1 - 6 and adds it to a running total.
* If the total has been reached or passed, the program prints how many rolls it took to reach the total.
* The program repeats the experiment a number of times specified by the user.
* At the end of all the simulations, the program prints the average number of rolls it took to reach the total number.
* The program prevents the user from typing invalid values and handles errors appropriately.
* The program only moves forward if input is valid.

## Spicy 🌶️🌶️🌶️
Create a program that satisfies the following requirements:

* The user inputs a possible die result between 1 and 6.
* The user inputs how many of that result they want.
* The user inputs how many times the simulation should run.
* The program randomly chooses an integer between 1 - 6, keeping track of how many times it selected number has come up.
* When the selected result has been rolled the specific number of times, the current iteration of the simulation ends and prints how many rolls it took.
* The simulation repeats the specified number of times.
* After all iterations of the program have finished, the program prints the average number of rolls it took in each simulation.
* The program prevents the user from typing invalid values and handles errors appropriately.
* The program only moves forward if input is valid.
