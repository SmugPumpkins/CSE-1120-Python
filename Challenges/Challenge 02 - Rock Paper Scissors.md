# Challenge 02 - Rock Paper Scissors
The object of this challenge is to create a rock paper scissors game for a user to play with the program.

## Mild 🌶️
Create a program that satisfies the following requirements:

* The program randomly chooses "rock", "paper", or "scissors".
* The user inputs "rock", "paper", or "scissors".
* The program checks if the player or the program won, or if there was a tie, and prints an appropriate message.
* The program prevents the user from typing invalid values and handles errors appropriately.
* The program only moves forward if input is valid.

## Medium 🌶️🌶️
Create a program that satisfies the following requirements:

* The program randomly chooses "rock", "paper", or "scissors".
* The user inputs "rock", "paper", or "scissors".
* The program checks if the player or the program won, or if there was a tie, and prints an appropriate message.
* The program prevents the user from typing invalid values and handles errors appropriately.
* After each game, the program asks the player if they would like to play again and repeats the process if so.
* The program only moves forward if input is valid.


## Spicy 🌶️🌶️🌶️
Create a program that satisfies the following requirements:

* The program randomly chooses "rock", "paper", or "scissors".
* The user inputs "rock", "paper", or "scissors".
* The program checks if the player or the program won, or if there was a tie and prints an appropriate message.
* The game continues until the program pr the player have won a best of 3 match (minimum of 2 wins).
* The program displays the winner of the best of 3 match and the number of games it took for the match to end.
* The program prevents the user from typing invalid values and handles errors appropriately.
* The program only moves forward if input is valid.
